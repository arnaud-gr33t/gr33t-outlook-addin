Outlook Calendar — Rendu des événements

Possibilités de différenciation visuelle des événements au sein d'un même calendrier Outlook, appliquées au calendrier overlay Gr33t Recovery.
Référence : Microsoft Graph API v1.0 — POST /me/calendars/{id}/events

1. showAs — Bande latérale gauche

Propriété la plus impactante visuellement. Contrôle la bande colorée sur le bord gauche de chaque événement dans la grille Semaine/Jour. 5 valeurs possibles :

free
Pas de bande
tentative
Hachures diag.
busy
Bande pleine
oof
Bande mauve
workingElsewhere
Pointillés
ValeurRenduUtilisation Gr33t
free Fond léger, pas de bande ⚡ Score all-day
tentative Bande hachurée diagonale ✅ Focus blocks
busy Bande pleine (couleur du calendrier) — non utilisé
oof Bande violette/mauve pleine 🌙 Overtime
workingElsewhere Bande en pointillés — non utilisé
// Graph API — création d'un événement focus
{
  "subject": "✅ Focus (45min)",
  "showAs": "tentative",
  "isReminderOn": false
}

2. categories — Pastilles de couleur

Chaque événement peut porter une ou plusieurs catégories avec un nom et une couleur prédéfinie. Outlook affiche une pastille colorée à côté du sujet (surtout en vue Liste et dans le popover). En vue Semaine/Jour, l'effet est plus subtil.

25 couleurs prédéfinies

Red
Orange
Brown
Yellow
Green
Teal
Olive
Blue
Purple
Cranberry
Steel
DarkSteel
Gray
DarkGray
Black
Prérequis : les catégories doivent d'abord être créées sur la boîte aux lettres via POST /me/outlook/masterCategories, sinon elles sont ignorées silencieusement par Graph.
// Créer une catégorie sur la boîte aux lettres
POST /me/outlook/masterCategories
{ "displayName": "Bon score", "color": "preset9" }

// Assigner à un événement
{ "categories": ["Bon score", "Green category"] }

Utilisation Gr33t potentielle : catégoriser les score events par score band (vert = bon, orange = modéré, rouge = faible) pour ajouter un indicateur visuel supplémentaire dans les vues Liste.

3. isAllDay — Positionnement

Contrôle si l'événement apparaît dans le bandeau all-day (en haut de chaque colonne) ou dans la grille horaire (positionné par start/end).

ValeurRenduUtilisation Gr33t
true Bandeau horizontal en haut du jour ⚡ Score quotidien
false Bloc dans la grille horaire ✅ Focus 🌙 Overtime

4. importance — Indicateur discret

Ajoute un petit indicateur visuel (icône) sur l'événement. Effet subtil, surtout visible en vue Liste.

ValeurRenduUsage potentiel
high❗ Flèche rouge / point d'exclamationJour en alerte (score < 50%)
normalRien (défaut)—
low↓ Flèche basse—
{ "importance": "high" }

5. sensitivity — Marquage de confidentialité

Contrôle la visibilité des détails pour les autres utilisateurs. Peu utile pour de la différenciation visuelle interne.

ValeurEffet
normalAucun marquage
personalLabel "Personnel"
private🔒 Détails masqués pour les tiers
confidentialSimilaire à private

6. subject — Emoji en préfixe

Le texte du sujet est affiché directement dans la grille calendrier. Les emoji rendent bien sur Outlook Web et Desktop — c'est notre principal levier de différenciation visuelle aujourd'hui.

⚡ Score quotidien
✅ Focus block
🌙 Overtime

Autres emoji utilisables :

🔴 Alerte
🟢 OK
🟡 Attention
⏳ Temps protégé
🧘 Concentration
📊 Rapport

7. body — Contenu HTML (popover uniquement)

Le corps de l'événement est en HTML et s'affiche dans le popover (clic sur l'événement) ou dans la vue détail. Aucun impact sur le rendu dans la grille calendrier.

Supporte les styles inline, tableaux HTML, couleurs — format "email HTML" (pas de CSS externe, pas de JavaScript). Gr33t l'utilise pour afficher le détail des 4 facteurs dans le popover du score event.

8. Couleur du calendrier

Contrainte majeure : un calendrier = une seule couleur pour tous ses événements. Le calendrier Gr33t Recovery est configuré en lightGreen. Il n'est pas possible d'avoir un événement rouge et un événement bleu dans le même calendrier — seuls showAs et categories permettent de différencier.

Couleurs de calendrier disponibles :

lightBlue
lightGreen
lightPurple
lightRed
lightYellow
lightTeal
lightGray
auto

Leviers inexploités les plus intéressants pour Gr33t

  • categories — code couleur par score band (vert/orange/rouge) sur les score events
  • busy et workingElsewhere — deux variantes visuelles showAs encore disponibles
  • importance: high — signaler un jour en alerte (score < 50%)
